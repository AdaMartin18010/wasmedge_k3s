## 🔍 收集方法

### 1. 官方文档收集

**收集步骤**：

1. 访问官方文档网站
2. 查找性能基准测试章节
3. 记录性能数据和测试环境
4. 标注数据来源和日期

**收集来源**：

- [WasmEdge 官方文档](https://wasmedge.org/docs/)
- [OPA 官方文档](https://www.openpolicyagent.org/docs/)
- [Gatekeeper 官方文档](https://open-policy-agent.github.io/gatekeeper/)
- [K3s 官方文档](https://docs.k3s.io/)

### 2. GitHub Issues 收集

**收集步骤**：

1. 访问 GitHub Issues 页面
2. 搜索性能相关议题
3. 查找性能测试结果
4. 记录测试环境和数据

**收集来源**：

- [WasmEdge GitHub Issues](https://github.com/WasmEdge/WasmEdge/issues)
- [OPA GitHub Issues](https://github.com/open-policy-agent/opa/issues)
- [K3s GitHub Issues](https://github.com/k3s-io/k3s/issues)

### 3. 技术会议演讲收集

**收集步骤**：

1. 查找 KubeCon、CNCF 等技术会议演讲
2. 查找性能基准测试演讲
3. 记录演讲中的性能数据
4. 标注演讲日期和来源

**收集来源**：

- KubeCon 2025 演讲
- CNCF 技术会议
- 社区技术分享

### 4. 社区博客文章收集

**收集步骤**：

1. 查找社区博客文章
2. 查找性能测试文章
3. 记录文章中的性能数据
4. 验证数据来源和测试环境

**收集来源**：

- 社区博客文章
- 技术博客
- 案例分享
