## 可观测性关系

```text
完整可观测性 = OTLP(横向) + eBPF(纵向)
OTLP提供横向坐标(请求链)
eBPF提供纵向坐标(内核栈)
```

## 可观测性优势分析

### 三大支柱

- **Metrics**：Prometheus（时间序列指标）
- **Logs**：Loki（日志聚合）
- **Traces**：Jaeger（分布式追踪）

### 横纵耦合定位

- **横向坐标（OTLP）**：请求链追踪，服务间调用关系
- **纵向坐标（eBPF）**：内核栈追踪，系统调用层级

### 关联能力

- **OTLP**：✅ 完整关联（Trace、Metrics、Logs 统一标签）
- **eBPF**：✅ 完整关联（内核层到应用层全链路）
- **传统工具**：⚠️ 部分关联（需要手动配置）

**优势**：OTLP + eBPF 组合提供最完整的可观测性能力
