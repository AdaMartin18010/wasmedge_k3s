## 12.4.3.1 结构主义选型格言

### 核心格言

> **"你要多假的机器？"**

- 想要**完全假的机器**→ 虚拟化（计算结构爆表）
- 想要**差不多假的机器**→ 容器化（计算-控制折中）
- 只想**别让程序坏事**→ 沙盒化（信息结构优先）

### 结构重心分析

| 需求         | 结构重心  | 技术选择           |
| ------------ | --------- | ------------------ |
| **完全隔离** | 计算结构  | 虚拟化             |
| **快速启动** | 信息结构  | 容器化/沙盒化      |
| **最小权限** | 信息结构  | 沙盒化             |
| **高密度**   | 信息结构  | 容器化/沙盒化      |
| **跨平台**   | 三轴平衡  | WASM 沙盒          |
| **强安全**   | 计算+信息 | gVisor/Firecracker |

## 12.4.3.2 选型/迁移口诀（结构版）

### 核心口诀

> **"向下买计算，向上买控制，向边买信息"**

### 详细解释

- **向下买计算**：

  - 需要**指令级兼容**→**往下走**：KVM/TEE
  - 通过硬件虚拟化获得计算结构保证

- **向上买控制**：

  - 需要**毫秒级热迁移**→**控制轴加码**：Firecracker 微 VM
  - 通过调度和同步机制优化控制结构

- **向边买信息**：
  - 需要**镜像秒级分发**→**信息轴优化**：Docker 分层+压缩
  - 需要**最小攻击面**→**信息轴剪枝**：seccomp + unveil + 线性内存

### 迁移策略

| 当前技术 | 目标需求 | 迁移方向     | 新技术选择         |
| -------- | -------- | ------------ | ------------------ |
| 虚拟化   | 更快启动 | 信息结构优化 | Firecracker        |
| 容器化   | 更强隔离 | 计算结构增强 | gVisor/Firecracker |
| 容器化   | 最小权限 | 信息结构剪枝 | seccomp + unveil   |
| 沙盒化   | 跨平台   | 三轴平衡     | WASM               |

## 12.4.3.3 选型决策矩阵

### 决策矩阵

| 需求维度     | 虚拟化 | 容器化 | 沙盒化 | gVisor | Firecracker | WASM  |
| ------------ | ------ | ------ | ------ | ------ | ----------- | ----- |
| **隔离级别** | ★★★★★  | ★★★    | ★★     | ★★★★   | ★★★★        | ★★★   |
| **启动速度** | ★★     | ★★★★   | ★★★★★  | ★★★    | ★★★★★       | ★★★★★ |
| **资源开销** | ★★     | ★★★★   | ★★★★★  | ★★★    | ★★★★        | ★★★★★ |
| **跨平台**   | ★      | ★★     | ★★     | ★      | ★           | ★★★★★ |
| **兼容性**   | ★★★★★  | ★★★★   | ★★★    | ★★★    | ★★★         | ★★    |
| **安全级别** | ★★★★★  | ★★★    | ★★★★   | ★★★★   | ★★★★        | ★★★★  |

### 场景选型

| 场景             | 推荐技术           | 理由                       |
| ---------------- | ------------------ | -------------------------- |
| **银行核心系统** | 虚拟化             | 需要硬件级隔离和热迁移     |
| **CI/CD 流水线** | 容器化             | 快速启动、高密度、成本敏感 |
| **插件系统**     | 沙盒化             | 最小权限、快速启动         |
| **边缘计算**     | WASM/Firecracker   | 快速启动、跨平台、资源受限 |
| **多租户 SaaS**  | gVisor/Firecracker | 强隔离、快速启动、资源效率 |
