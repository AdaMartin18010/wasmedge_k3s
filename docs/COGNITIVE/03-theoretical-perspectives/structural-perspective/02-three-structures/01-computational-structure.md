## 12.2.1.1 定义与核心特征

### 定义

计算结构关注的是**"什么可以被计算"**以及**"如何通过规则组合出复杂行为"**。这是计
算机科学的"语法"层面。

### 类比

代数结构（群、环、幺半群）

### 核心特征

- **封闭性**：操作作用于对象后仍在系统内（如函数组合）
- **结合律**：计算顺序可重组（如函数复合、Monad 结合律）
- **单位元**：空操作、恒等函数、return/unit
- **可组合性**：小部件可组合成大系统（模块化、函数式编程）

### 与代数结构的对应

| 代数结构特征 | 计算结构对应                   | 示例                        |
| ------------ | ------------------------------ | --------------------------- |
| **封闭性**   | 函数组合仍在函数空间内         | `f . g :: a -> c`           |
| **结合律**   | 函数复合满足结合律             | `(f . g) . h = f . (g . h)` |
| **单位元**   | 恒等函数 `id`                  | `id . f = f . id = f`       |
| **逆元**     | 在某些情况下存在（如可逆函数） | `f . f⁻¹ = id`（如果存在）  |

## 12.2.1.2 典型实例

### 1. λ-演算

**结构**：代数系统

**代数解释**：

- 变量绑定 → 抽象（λ-abstraction）
- 应用 → 二元运算（function application）
- β-归约 → 重写规则（rewrite rule）

**示例**：

```lambda
λx. x + 1        -- 抽象（单位元）
(λx. x + 1) 2   -- 应用（二元运算）
→ 2 + 1         -- β-归约（重写规则）
→ 3
```

### 2. 代数数据类型（ADT）

**结构**：初始代数

**代数解释**：

- `List a = Nil | Cons a (List a)` 是函子 `F(X) = 1 + a × X` 的初始代数
- `1` 表示空列表（Nil）
- `a × X` 表示 `Cons a (List a)`

**示例**：

```haskell
-- 函子定义
data ListF a x = NilF | ConsF a x

-- List 是 ListF 的初始代数
data List a = Nil | Cons a (List a)

-- 初始性：存在唯一的态射
foldr :: (a -> b -> b) -> b -> List a -> b
```

### 3. Monad

**结构**：幺半群

**代数解释**：

- `join :: M (M a) → M a` 是乘法
- `return :: a → M a` 是单位元
- 满足结合律：`join . join = join . fmap join`
- 满足单位律：`join . return = id` 和 `join . fmap return = id`

**示例**：

```haskell
class Monad m where
  return :: a -> m a           -- 单位元
  (>>=)  :: m a -> (a -> m b) -> m b  -- 乘法（通过 join 定义）

-- 结合律
(m >>= f) >>= g = m >>= (\x -> f x >>= g)

-- 单位律
return x >>= f = f x
m >>= return = m
```

### 4. 图灵机

**结构**：状态变换系统

**代数解释**：

- 状态转移函数：`δ: Q × Γ → Q × Γ × {L,R}`
- 构成状态空间上的"作用"（action）
- 状态序列构成计算路径

**示例**：

```text
状态空间：Q = {q₀, q₁, q₂, ...}
字母表：Γ = {0, 1, B}
转移函数：δ(q₀, 1) = (q₁, 0, R)
```

## 12.2.1.3 形式化工具

### 1. 范畴论

**应用**：可用笛卡尔闭范畴建模 λ-演算

**核心概念**：

- **对象**：类型
- **态射**：函数
- **笛卡尔闭**：存在积和指数对象

**示例**：

```text
类型 a 和 b 的积：a × b
类型 a 和 b 的指数：a → b（函数类型）
```

### 2. 幺半群

**应用**：Monad 是范畴上的幺半群

**核心概念**：

- **对象**：类型构造子 `M`
- **乘法**：`join :: M (M a) → M a`
- **单位元**：`return :: a → M a`

### 3. 初始代数

**应用**：代数数据类型是初始代数

**核心概念**：

- **函子**：`F(X) = 1 + a × X`
- **初始代数**：存在唯一的态射到任何其他代数

## 12.2.1.4 哲学意义

计算结构关注的是**"什么可以被计算"**以及**"如何通过规则组合出复杂行为"**。这是计
算机科学的"语法"层面。

### 核心洞察

1. **可组合性**：小部件可组合成大系统
2. **规则性**：复杂行为由简单规则组合而成
3. **抽象性**：计算结构独立于具体实现

### 设计原则

- **模块化**：将复杂系统分解为可组合的小部件
- **函数式编程**：强调函数的组合和变换
- **类型系统**：通过类型确保组合的正确性
