## 1. 标注目的

**案例真实性标注**的目的是：

- ✅ 明确案例的来源和真实性
- ✅ 帮助读者判断案例的可靠性
- ✅ 区分真实生产环境案例和理论验证案例
- ✅ 提供案例验证方法

---

## 2. 案例类型分类

### 2.1 真实案例（Real Production Case）

**定义**：来自真实生产环境的案例，包含实际部署数据、性能指标和验证结果。

**特征**：

- 来自真实企业或组织的生产环境
- 包含实际部署数据（节点数、Pod 数、QPS 等）
- 包含实际性能指标（延迟、吞吐量、资源占用等）
- 包含验证方法和验证结果
- 可以标注来源（企业名称、项目名称等，如果允许）

**标注**：`✅ 真实案例`

### 2.2 理论案例（Theoretical Case）

**定义**：基于理论分析和最佳实践构建的案例，用于验证技术方案的可行性。

**特征**：

- 基于技术理论和最佳实践构建
- 包含假设条件和适用场景
- 包含理论性能指标（基于技术特性估算）
- 用于验证技术方案的可行性
- 不来自真实生产环境

**标注**：`📊 理论案例`

### 2.3 混合案例（Hybrid Case）

**定义**：结合真实案例和理论分析的案例，部分数据来自真实环境，部分数据来自理论分析。

**特征**：

- 部分数据来自真实生产环境
- 部分数据来自理论分析
- 明确标注哪些数据是真实的，哪些是理论的
- 用于展示技术方案的完整应用场景

**标注**：`🔀 混合案例`

## 3. 标注规范

### 3.1 标注位置

**在案例文档的"案例基本信息"部分添加**：

```markdown
## 1. 📋 案例基本信息

**案例名称**：XXX

**行业**：XXX

**场景**：XXX

**案例类型**：✅ 真实案例 / 📊 理论案例 / 🔀 混合案例

**来源**：XXX

**验证状态**：✅ 已验证 / ⚠️ 待验证

**验证方法**：XXX（如果是真实案例，说明验证方法）

**收集日期**：YYYY-MM-DD
```

### 3.2 标注内容

**真实案例必须包含**：

- 案例类型：✅ 真实案例
- 来源：企业名称、项目名称（如果允许）或"基于 XXX 企业生产环境"
- 验证方法：实际部署验证、性能测试、生产环境监控等
- 验证结果：实际性能指标、部署规模等

**理论案例必须包含**：

- 案例类型：📊 理论案例
- 来源：基于技术理论和最佳实践
- 假设条件：明确说明假设条件
- 适用场景：说明适用场景和限制

**混合案例必须包含**：

- 案例类型：🔀 混合案例
- 来源：部分数据来自真实环境，部分数据来自理论分析
- 数据标注：明确标注哪些数据是真实的，哪些是理论的

## 4. 标注示例

### 4.1 真实案例示例

```markdown
## 1. 📋 案例基本信息

**案例名称**：支付网关系统边缘部署

**行业**：金融

**场景**：边缘计算、Serverless、容器化

**案例类型**：✅ 真实案例

**来源**：基于某大型支付机构生产环境（已脱敏）

**验证状态**：✅ 已验证

**验证方法**：
- 实际部署验证：在 50+ 边缘节点部署
- 性能测试：P99 延迟 < 50ms，QPS 10,000+
- 生产环境监控：99.99% 可用性

**收集日期**：2025-11-07
```

### 4.2 理论案例示例

```markdown
## 1. 📋 案例基本信息

**案例名称**：边缘 AI 推理系统

**行业**：AI/ML

**场景**：边缘计算、AI 推理

**案例类型**：📊 理论案例

**来源**：基于边缘 AI 推理技术理论和最佳实践

**假设条件**：
- 边缘节点资源：4C8G
- 模型大小：< 100MB
- 网络带宽：100Mbps

**适用场景**：
- 边缘 AI 推理场景
- 资源受限的边缘环境
- 需要低延迟推理的场景

**验证状态**：⚠️ 待验证

**收集日期**：2025-11-07
```

### 4.3 混合案例示例

```markdown
## 1. 📋 案例基本信息

**案例名称**：大规模 Serverless 平台

**行业**：云计算

**场景**：Serverless、容器化

**案例类型**：🔀 混合案例

**来源**：
- 真实数据：基于某云服务提供商的部分生产数据（已脱敏）
- 理论数据：基于 Serverless 架构理论分析

**数据标注**：
- **真实数据**：部署规模（10+ 节点，5000+ Pod）、性能指标（冷启动 < 1ms）
- **理论数据**：成本优化数据（基于理论分析估算）

**验证状态**：✅ 部分验证（真实数据已验证）

**收集日期**：2025-11-07
```

## 5. 标注检查清单

### 5.1 标注完整性检查

- [ ] 案例类型已标注（✅ 真实案例 / 📊 理论案例 / 🔀 混合案例）
- [ ] 来源已说明
- [ ] 验证状态已标注
- [ ] 验证方法已说明（如果是真实案例）
- [ ] 假设条件已说明（如果是理论案例）
- [ ] 数据标注已说明（如果是混合案例）

### 5.2 标注准确性检查

- [ ] 案例类型与内容一致
- [ ] 来源信息准确
- [ ] 验证方法合理
- [ ] 假设条件明确
- [ ] 数据标注清晰
